Выполни следующие задачи строго по шагам:

$ARGUMENTS

---

## 1. Анализ аргументов
Проведи первичный анализ того, что требуется протестировать:
- назначение, цели и предполагаемое поведение,
- возможные входы и выходы,
- критические и уязвимые зоны,
- ограничения и скрытые предположения,
- потенциальные точки отказа,
- публичный интерфейс и допустимые способы взаимодействия.

---

## 2. План тестирования
Сформируй подробный, технически строгий план тестирования, который должен включать:

### 2.1 Функциональные тесты
- основные сценарии работы,
- корректные данные,
- пограничные условия,
- тесты на соответствие спецификации.

### 2.2 Негативные тесты
- некорректные входы (тип, размер, формат, диапазон),
- отсутствие данных,
- конфликтующие данные,
- случайно сгенерированные входы,
- сценарии неправильного использования.

### 2.3 Black-box тестирование
- тестирование строго по интерфейсу,
- без предположений о внутренней реализации,
- проверка инвариантов,
- проверка устойчивости поведения.

### 2.4 Попытки «сломать» систему
- стресс-тесты,
- fuzz-тестирование,
- экстремальные сценарии,
- неожиданные входы,
- попытки вызвать сбои или неконсистентное поведение.

### 2.5 Устойчивость и стабильность
- тесты на повторяемость,
- оценка стабильности при множественных вызовах,
- тесты на деградацию,
- проверка корректности в условиях большого объёма данных.

---

## 3. Создание нескольких наборов тестов
Сгенерируй **не менее трёх равнозначных наборов тестов**, каждый:
- использует разные подходы и стратегические принципы тестирования,
- является самодостаточным,
- не содержит ни прямых, ни косвенных указаний, какой из вариантов лучше.

---

## 4. Контекст и объяснение
Добавь расширенный контекст:
- зачем такие тесты важны,
- какие риски они минимизируют,
- почему предложенные методы покрывают проблему,
- критерии качества и глубины тестового покрытия.

---

## 5. Анализ через несколько LLM
Передай *всю итоговую структуру* (анализ, план, наборы тестов, контекст) в следующие модели:

- **codex-cli 5.1**
- **gemini-cli**
- **claude-cli**

Каждая модель должна:

- применить критическое мышление,
- поставить под сомнение корректность каждого набора тестов,
- выявить логические ошибки, недочёты и слабые места,
- анализировать непредвзято,
- использовать этическое мышление (влияние, риски, злоупотребления),
- применить лучшие практики тестирования и ревью,
- оценить полноту покрытия, устойчивость и разнообразие сценариев,
- попытаться предсказать, какие тесты наиболее вероятно найдут реальные ошибки.

Каждая модель обязана вернуть:

1. **замечания по каждому набору тестов**,  
2. **точечные рекомендации по улучшению**,  
3. **оценку надёжности и глубины покрытия**,  
4. **рейтинг каждого варианта по шкале 1–10**,  
5. **указание, какой набор она считает предпочтительным, и почему**.

---

## 6. Финальный сводный вывод
На основе анализа трёх моделей создай итоговую сводку:

- сравнение рейтингов по всем вариантам,
- сильные и слабые стороны каждого набора тестов,
- агрегированная оценка устойчивости,
- определение, какой вариант объективно выглядит наиболее надёжным с учётом мульти-LLM анализа,
- общие рекомендации по улучшению финального тестового набора,
- краткий вывод о полноте покрытия относительно $ARGUMENTS.

---

## 7. Формат вывода
Выводи результат строго в структурированном формате:
- уровневые заголовки,
- подзаголовки,
- списки,
- подсписки,
- таблицы (при необходимости),
- отдельные блоки для тестов.

Не пропускай ни одного шага.
